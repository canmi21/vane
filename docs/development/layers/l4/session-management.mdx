## UDP Session Management

Since UDP is connectionless, Vane creates **Virtual Sessions** to maintain stateful flows (e.g., for NAT-like behavior).

<Mermaid
	chart="
graph LR
    Client -->|Packet A| Vane

    Vane -- Lookup Session --> Client

/>

- **Key**: `(Client IP, Protocol)`
- **Timeout**: `UDP_SESSION_TIMEOUT_SECS` (Default: 30s).
- **Cleanup**: A background task prunes expired sessions to prevent memory leaks.

## Health Checking

Vane maintains a health registry to avoid routing to dead backends.

### TCP (Active)

Vane proactively connects to TCP targets.

- **Interval**: `HEALTH_TCP_INTERVAL_SECS` (Default: 5s).
- **Timeout**: `HEALTH_TCP_CONNECT_TIMEOUT_MS` (Default: 2000ms).
- **Recovery**: Automatic when a probe succeeds.

### UDP (Passive)

Since UDP is connectionless, Vane relies on **Passive Monitoring**.

- **Failure**: If an IO error occurs during forwarding, the target is marked "Unhealthy".
- **Recovery**: The target remains marked for `HEALTH_UDP_UNHEALTHY_TTL_SECS` (Default: 10s), then is tentatively retried.
