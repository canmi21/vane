	Legacy Routing is provided for strict backward compatibility. It will not receive new features,
	but it will not be deprecated. It functions similarly to traditional Reverse Proxy Virtual Hosts.
	For modern use cases and advanced capabilities, **Flow Routing** is the recommended primary path.
</Callout>

## Overview

Legacy Routing provides strict backward compatibility with older Vane configurations. It uses a linear list of prioritized rules.

**Source**: `src/layers/l4/legacy/`

## Logic

<Steps>
<Step>

### Sorting

Vane loads the protocol rules from `ports.json` and sorts them by their `priority` field (lower numbers execute first).

<Mermaid
	chart="
graph LR
    Config[Load Config] --> Rules[List of Rules]
    Rules --> Sort[Sort by Priority]
    Sort --> Sorted[Ordered Rules]
"
/>

</Step>
<Step>

### Peeking

To match rules, Vane peeks at the initial bytes of the connection without consuming them.

<Mermaid
	chart="
graph LR
    Socket[Socket] --> Peek[Peek N Bytes]
    Peek --> Buffer[Peek Buffer]
"
/>

</Step>
<Step>

### Matching

The engine iterates through the sorted rules, testing each `detect` pattern against the peek buffer.

<Mermaid
	chart="
graph LR
    Buffer --> Rule1{Match Rule 1?}
    Rule1 -- No --> Rule2{Match Rule 2?}
    Rule1 -- Yes --> Dispatch[Execute Target]
    Rule2 -- Yes --> Dispatch
"
/>

</Step>
<Step>

### Target Dispatch

Once a match is found, the connection is either forwarded to a target pool or dropped if no rules match.

<Mermaid
	chart="
graph LR
    Match[Match Found] --> Target{Target Type}
    Target -- Forward --> Proxy[L4 Forwarder]
    Target -- Resolver --> Legacy[Legacy Resolver]
"
/>

</Step>
</Steps>

## Detection Methods

| Method         | Description                                                    |
| :------------- | :------------------------------------------------------------- |
| **`magic`**    | Matches hex byte patterns (e.g., `0x1603`).                    |
| **`prefix`**   | Matches string prefixes (e.g., `GET /`).                       |
| **`regex`**    | Regular expression matching (Requires `tcp` or `udp` feature). |
| **`fallback`** | Always matches (Catch-all).                                    |

> **Note**: Legacy routing does **not** support Upgrades or Layer transitions. It is purely for L4 forwarding.
