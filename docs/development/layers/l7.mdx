Layer 7 (L7) is where Vane performs full **HTTP Termination**. Unlike L4/L4+, it parses the entire request (Headers & Body), allowing for deep inspection, transformation, and synthetic responses.

## The Container

At the heart of L7 is the `Container`. It is the universal envelope for all requests.

```rust
pub struct Container {
    // Control Plane (Metadata for Routing)
    pub kv: KvStore,

    // Data Plane (The actual HTTP content)
    pub request_headers: HeaderMap,
    pub request_body: PayloadState,
    pub response_headers: HeaderMap,
    pub response_body: PayloadState,

    // Extension Plane (Protocol-specific hooks)
    pub protocol_data: Option<Box<dyn ProtocolData>>,
}
```

### Payload State (Lazy Buffering)

Vane aims to be zero-copy where possible. The body uses a lazy state machine:

| State | Description |
| :--- | :--- |
| **`Http`** | A raw stream from the socket (Hyper Body). Low memory footprint. |
| **`Buffered`** | Fully loaded into memory (`Bytes`). Triggered by accessing `{{req.body}}`. |
| **`Empty`** | No body exists. |

> **Memory Safety**: Buffering is protected by a global `AtomicUsize` counter. If the system runs low on memory, Vane will reject buffering attempts to prevent OOM (Out Of Memory) crashes.

## The HTTP Adapter

The **Adapter** (`httpx.rs`) bridges the gap between the underlying HTTP Engine (Hyper) and Vane's Flow Engine.

### Lifecycle

<Steps>
<Step>
### Ingress
Hyper accepts the connection (H1, H2, or H3) and parses the request headers.
</Step>

<Step>
### Injection
The Adapter populates the `Container`:
- **KV**: `req.method`, `req.path`, `req.host`, `req.proto`.
- **Headers**: Moved zero-copy into `request_headers`.
- **Body**: Wrapped in `PayloadState::Http`.
</Step>

<Step>
### Execution
The Flow Engine executes middleware.
- **Middleware** can inspect headers or force-buffer the body.
- **Drivers** (like `static-files`) populate `response_body`.
</Step>

<Step>
### Termination
A Terminator (e.g., `send_response`) signals completion.
- It sends the `response_headers` via a oneshot channel.
- It yields the `response_body` back to Hyper for streaming to the client.
</Step>
</Steps>

## WebSocket Support

WebSockets require a "Protocol Upgrade" within L7.

1.  **Handshake**: The Adapter detects `Upgrade: websocket`.
2.  **Capture**: It captures the `OnUpgrade` future from Hyper.
3.  **Storage**: The handle is stored in `container.protocol_data`.
4.  **Terminator**: The `response` terminator detects the handle, sends a `101 Switching Protocols` response, and establishes a bidirectional tunnel.
