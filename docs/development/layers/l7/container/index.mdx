## Overview

At the heart of L7 is the `Container`. It is the universal envelope for all requests, providing a unified interface for middleware regardless of the underlying protocol.

**Source**: `src/layers/l7/container.rs`

## Structural Components

<Steps>
<Step>

### The Three Planes

The Container is organized into three distinct logical planes to separate metadata, content, and protocol-specific extensions.

<Mermaid
	chart="
graph LR
    subgraph Container
        KV[Control Plane: KV Store]
        DP[Data Plane: Headers/Body]
        EP[Extension Plane: Protocol Data]
    end
"
/>

- **Control Plane**: Metadata for routing and logic (`conn.ip`, `req.path`).
- **Data Plane**: The actual HTTP payload (Headers and Body).
- **Extension Plane**: Protocol-specific hooks (WebSocket handles).

</Step>
<Step>

### Payload State Machine

Vane uses a lazy state machine for request and response bodies to achieve zero-copy performance.

<Mermaid
	chart="
graph LR
    Incoming[Socket Bytes] --> Http[PayloadState::Http]
    Http -->|Middleware Access| Transition{Force Buffer?}
    Transition -- No --> Stream[Streaming to Upstream]
    Transition -- Yes --> Buffered[PayloadState::Buffered]
"
/>

- **`Http`**: A raw stream (low memory).
- **`Buffered`**: Fully loaded into memory (after template resolution or middleware request).

</Step>
<Step>

### Memory Safety Layer

To prevent resource exhaustion (OOM), every buffering operation is guarded by a two-tier verification system.

<Mermaid
	chart="
graph LR
    Req[Buffer Request] --> PerReq{Per-Request Limit?}
    PerReq -- Exceeded --> Fail([System Error])
    PerReq -- OK --> Global{Global RAM Quota?}
    Global -- OK --> Guard[BufferGuard Created]
    Global -- Full --> Fail
"
/>

1.  **Global Limit**: `L7_ADAPTIVE_MEMORY_LIMIT` (Default 85% RAM).
2.  **Per-Request**: `L7_MAX_BUFFER_SIZE` (Default 10MB).

</Step>
</Steps>

## Implementation

```rust
pub struct Container {
    // Control Plane (Metadata for Routing)
    pub kv: KvStore,

    // Data Plane (The actual HTTP content)
    pub request_headers: HeaderMap,
    pub request_body: PayloadState,
    pub response_headers: HeaderMap,
    pub response_body: PayloadState,

    // Extension Plane (Protocol-specific hooks)
    pub protocol_data: Option<Box<dyn ProtocolData>>,
}
```
