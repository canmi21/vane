## Overview

TCP handling in Vane acts as the primary ingress for stream-based traffic. It supports two modes of operation: **Legacy** (Rule-based) and **Flow** (Plugin-based).

**Source**: `src/layers/l4/tcp.rs`

## Peeking Mechanism

To route traffic without consuming it (essential for TLS SNI routing), Vane "peeks" at the initial bytes of the connection.

- **Config**: `TCP_DETECT_LIMIT` (Default: 64 bytes).
- **Max**: 8192 bytes.
- **Behavior**: If the peek fails or returns 0 bytes, the connection is closed.

## Modes

### Flow Mode (Recommended)

Delegates control to the [Flow Engine](./../../../engine/flow-executor).

- Populates `TransportContext`.
- Allows complex branching and upgrades (e.g., `tcp -> tls -> http`).

### Legacy Mode

Uses a static list of rules matched by priority.

- Supports `magic`, `prefix`, and `regex` matching.
- Direct forwarding to target pools.
