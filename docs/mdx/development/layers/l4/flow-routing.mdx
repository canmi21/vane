## Overview

Flow Routing is the modern execution model for Vane. Instead of static rules, it uses a **Tree of Plugins**.

**Source**: `src/layers/l4/flow.rs`

## Execution Context

Flows operate within a `TransportContext`, which carries the connection metadata and peeked data across the plugin chain.

```rust
pub struct TransportContext {
    pub kv: KvStore,
    pub payloads: AHashMap<String, Bytes>, // Cached peek buffers
}
```

## Logic

<Steps>
<Step>

### Initialization

The `kv` store is populated with core metadata (`conn.ip`, `conn.port`, `conn.uuid`), and the `conn.layer` is set to `l4`.

<Mermaid
	chart="
graph LR
    In[New Conn] --> KV[Populate KV]
    KV --> Layer[Set layer=l4]
    Layer --> Context[Create TransportContext]
"
/>

</Step>
<Step>

### Recursive Execution

The `engine::executor` traverses the plugin tree defined in the port configuration.

<Mermaid
	chart="
graph LR
    Context --> Exec[Flow Executor]
    Exec --> Node1[Plugin A]
    Node1 -- Branch X --> Node2[Plugin B]
    Node1 -- Branch Y --> Node3[Plugin C]
"
/>

</Step>
<Step>

### Terminal Handling

The flow continues until a Terminator returns a final result.

<Mermaid
	chart="
graph LR
    Exec --> Term{Terminator}
    Term -- Finished --> Done([End Flow])
    Term -- Upgrade --> L4P([Handover to L4+])
"
/>

</Step>
</Steps>
