## Logic Flow

The TLS Inspector performs a "Smart Peek" operation to extract routing metadata without terminating the connection.

**Source**: `src/layers/l4p/tls.rs`

<Steps>
<Step>

### Smart Peek

The inspector peeks at the initial bytes of the TCP stream. It handles fragmentation by waiting until a full TLS record header is available.

<Mermaid
  chart="
graph LR
    Stream[TCP Stream] --> Peek{Peek Bytes}
    Peek -- < 5 bytes --> Wait[Wait More]
    Peek -- >= 5 bytes --> CheckLen{Check Record Len}
    CheckLen -- Partial --> Wait
    CheckLen -- Full --> Buffer[Capture Buffer]
"
/>

</Step>
<Step>

### Parsing & Extraction

The captured buffer is parsed to validate the `ClientHello` handshake and extract extensions.

<Mermaid
  chart="
graph LR
    Buffer[Record Bytes] --> Parser[ClientHello Parser]
    Parser --> Valid{Is Valid?}
    Valid -- Yes --> Extract[Extract SNI/ALPN]
    Valid -- No --> Error[Fail Open/Closed]
    Extract --> KV[(KV Store)]
"
/>

</Step>
<Step>

### Resolver Execution

With the SNI populated in the KV Store (`tls.sni`), the engine executes the configured TLS resolver flow to decide the next action.

<Mermaid
  chart="
graph LR
    KV[(KV Store)] --> Resolver[Load tls.json]
    Resolver --> Flow[Execute Flow]
    Flow --> Action{Result}
    Action -- Proxy --> L4Proxy[L4 Forwarding]
    Action -- Upgrade --> L7[Upgrade to L7]
"
/>

</Step>
</Steps>
