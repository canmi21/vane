## Overview

QUIC requires specialized handling due to its connectionless nature (UDP) but stateful requirements (Encryption/Streams).

<Steps>
<Step>

### Session Tracking

Vane tracks QUIC sessions primarily using the **Connection ID (CID)** to support connection migration.

<Mermaid
	chart="
graph LR
    Packet[UDP Packet] --> Extract[Extract DCID]
    Extract --> Lookup{CID Registry}
    Lookup -- Hit --> Action[Execute Cached Action]
    Lookup -- Miss --> L4[New L4 Flow]
"
/>

</Step>
<Step>

### Sticky IP Fallback

If CID lookup fails (e.g., during server-initiated CID migration or transparent proxying where CIDs change), Vane falls back to IP-based stickiness.

<Mermaid
	chart="
graph LR
    Lookup -- Miss --> Sticky{Check Sticky Map}
    Sticky -- Hit --> Forward[Forward to Upstream]
    Sticky -- Miss --> Drop[Drop or New Flow]
"
/>

</Step>
<Step>

### Buffer Safety

To prevent amplification attacks or memory exhaustion, Vane enforces strict limits on pending QUIC packets before they are accepted by a flow.

<Mermaid
	chart="
graph LR
    Packet --> Global{Global Limit?}
    Global -- OK --> Session{Session Limit?}
    Global -- Full --> Drop1[Drop Packet]
    Session -- OK --> Buffer[Buffer Packet]
    Session -- Full --> Drop2[Drop Packet]
"
/>

- **Global Limit**: `QUIC_GLOBAL_PENDING_BYTES_LIMIT` (Default: 64MB).
- **Session Limit**: `QUIC_SESSION_BUFFER_LIMIT` (Default: 64KB).

</Step>
</Steps>
