# crates/wasm/fixtures

This directory contains pre-built WebAssembly component fixtures used by integration tests.

## metadata_fixture.wasm

A minimal Component Model component generated by `build.rs` at compile time.

It exports:

- `vane:plugin/registry@0.1.0` — `get-metadata` returns a hardcoded `Metadata`
  value: name="fixture", version="0.1.0", abi-version="0.1.0", one l4-peek export
  named "probe" (stateless=true, needs-body=false, inspects=[]).
- `vane:plugin/handler-l4-peek@0.1.0` — `handle` is exported with the correct
  canonical ABI signature but traps unconditionally (unreachable). Presence of
  this export is enough for handler-kind validation tests.

The component has no host imports; the core module uses only static data and a
simple bump allocator for the canonical ABI return-pointer allocation.

## Generation

`build.rs` runs `wit-parser` + `wat` + `wit-component` at cargo build time:

1. Parse `wit/` to load the `vane:plugin` and `vane:host` interface definitions.
2. Inline a `fixture-plugin` world (exports only, no host imports).
3. Assemble the WAT core module via `wat::parse_str`.
4. Call `embed_component_metadata` to attach the `component-type` custom section.
5. Wrap with `ComponentEncoder` to produce the final component binary.

The file is regenerated whenever `build.rs` or the `wit/` directory changes.
